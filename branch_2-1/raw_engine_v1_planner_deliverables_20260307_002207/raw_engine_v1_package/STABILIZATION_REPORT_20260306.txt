2. planner oracle design
- reference planner uses the same make_basic_hooks_for_target() policy as the engine planner
- primitive semantics are composed from the existing reference isolate/split/join/integrate models
- compared signatures:
  - final full canonical raw state
  - target stop condition
  - target-occ prepare_isolate_neighborhood signature
  - debug trace summary when captured
- planner mismatch output now prefixes:
  - case
  - seed
  - iter
  - targetOcc
  - keepOcc
  - stepBudget

3. failure signature
- FailureClass:
  - none
  - validator_failure
  - primitive_oracle_mismatch
  - planner_oracle_mismatch
  - step_budget_exceeded
  - sanitizer_failure
  - crash
- FailureSignature:
  - failureClass
  - hash
  - detail
- replay_state and replay_planner_state print FAIL_CLASS=... and FAIL_HASH=... when present
- reducer acceptance now requires same failure class and uses hash equality when both sides have a hash

4. dump / replay / reduce

- planner_oracle_fuzz auto-reduces a pending planner dump when a dump-on-fail planner failure is hit

10. remaining risks
- planner trace dumping is intentionally verbose; large fuzz runs can create many trace files quickly
- reducer matching uses detail hashes from replay output, so message-format churn can invalidate hash stability across future refactors
- planner oracle compares final state / stop condition / target prepare, not strict trace equality
