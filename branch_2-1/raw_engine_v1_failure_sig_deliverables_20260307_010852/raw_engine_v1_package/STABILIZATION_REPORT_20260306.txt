- replay_state / replay_planner_state now print machine-readable FAIL_* lines from the structured object, including FAIL_HASH derived from structural fields only.

3. reducer acceptance rules
- reducer acceptance no longer uses detail-string equality.
- acceptance is FailureSignature::same_failure():
  - failureClass must match
  - stage / primitiveKind / plannerPhase / targetOcc / mismatchKind must match when both sides carry a concrete value
  - canonicalStateHash / oracleHash / tracePrefixHash must match when both sides are populated
- effect:
  - planner / primitive oracle mismatches keep the same structural mismatch class and canonical hashes
  - step_budget_exceeded can shrink budget without getting blocked by message-format churn
  - validator / sanitizer / crash classes are separated cleanly at replay time

4. stepBudget / trace shrink strategy
- planner reducer still shrinks raw state structurally:
  - keepOcc
  - occurrence removal
  - REAL vertex removal
  - edge removal
- added planner-specific shrink axes:
  - stepBudget
  - tracePrefixLength
  - initialQueue size
  - per-job boundary map size inside initialQueue
- reduced planner artifacts are now split:
  - reduced_planner/<stem>_reduced_planner_state.txt
  - reduced_planner/<stem>_reduced_planner_trace.log

8. artifact retention policy
- default artifact root remains executable-parent/artifacts/
- default deterministic roots:

  - build-asan/tests/artifacts/
  - build-release/tests/artifacts/
- subdirectories:
  - counterexamples/
  - reduced/
  - reduced_planner/

  - logs/
- policy:
  - successful cases do not keep traces by default
  - failure artifacts are kept
  - reduced / reduced_planner artifacts are pruned after traces / logs / counterexamples

10. remaining risks
- sampled planner fuzz traces are still isolate-heavy; the current generator/profile tuning changes artifact distribution strongly, but did not yet drive split/join/integrate hits in the 5x1000 release sweep.
- planner reducer now shrinks stepBudget / tracePrefix / initialQueue, but it still reduces planner behavior indirectly through raw-state deletion rather than a dedicated semantic queue minimizer.
- --compress-artifacts currently matters for log-like artifacts; replayable txt state dumps are intentionally kept uncompressed.
