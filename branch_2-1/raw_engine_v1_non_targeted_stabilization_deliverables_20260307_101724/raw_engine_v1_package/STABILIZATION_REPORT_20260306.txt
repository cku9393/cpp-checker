2. new generator family / bias profile summary
- non-targeted planner generation now resolves a precondition-bias config from:
  - weight profile defaults
  - optional `--precondition-bias-profile`
  - optional `--bias-split`, `--bias-join`, `--bias-integrate` overrides
- `random` / `weighted_*` / `artifact_heavy` planner fuzz no longer rely on the old isolate-heavy single-shape generator only
  - legacy single-skeleton generation is still in the pool
  - structural precondition states are now mixed in probabilistically, with weights controlled by the resolved bias config
  - this keeps non-targeted modes random, but materially raises split/join/integrate entry probability
- new direct structural families:
  - `join_ready`
    - builds two sibling skeletons from scratch with the same separator pair
    - planner starts from a direct `JOIN_PAIR` precondition state, not a post-split artifact
  - `integrate_ready`
    - builds a parent target skeleton plus a direct integrate child from scratch
    - child is either boundary-only or boundary-map style, selected deterministically from seed
  - `planner_mixed_structural`
    - deterministic rotation over `join_ready`, `integrate_ready`, `split_ready`, `split_with_boundary_artifact`
    - increases primitive-sequence diversity while allowing join/integrate before any planner-issued split
- existing targeted families were kept:
  - `split_ready`
  - `split_with_boundary_artifact`
  - `split_with_keepOcc_sibling`
  - `split_with_join_and_integrate`
  - `planner_mixed_targeted`

9. constraints preserved
- committed raw skeleton still uses only `REAL` / `OCC_CENTER`
- `ALLOC_LINK` is still only `RawOccRecord::allocNbr` metadata
- primitive bodies still use builder reconstruction plus `commit_skeleton()`
- only `afterSplit` constructs `JOIN_PAIR` / `INTEGRATE_CHILD`
- `afterJoin` / `afterIntegrate` still only guarantee `ENSURE_SOLE(targetOcc)` once
- `src/` primitive bodies were not changed in this phase
- support-based brute-force separator discovery remains unchanged

10. remaining risks
- non-targeted coverage is now materially better, but it is still bias-assisted; pure emergent random discovery from the legacy shape alone would still under-cover split/join/integrate
- join remains structurally special:
  - an empty-queue planner run cannot discover a fresh `JOIN_PAIR` from `ENSURE_SOLE(targetOcc)` alone under the current hook rules
  - current non-targeted join coverage therefore comes from probabilistic direct structural precondition states, while deterministic coverage comes from `join_ready` / `planner_mixed_structural`
- artifact and integrate coverage are now healthy, but conversion ratios are partly shaped by the builder pool design rather than by fully unconstrained random discovery
- warnings around existing unqualified `move` calls and some aggregate initialization remain in the tree; they are noisy but not functional regressions
