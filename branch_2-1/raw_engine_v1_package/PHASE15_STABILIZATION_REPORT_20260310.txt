3. compare eligibility stats definition
- `compare_eligible_state_count`: states that can actually run the split-choice exact compare path
- `compare_ineligible_state_count`: states sampled for compare/audit but rejected before exact compare
- `compare_ineligible_reason_histogram`: reason breakdown for ineligible states
- `compare_completed_state_count`: states that finished exact compare with exact audit available
- `compare_partial_state_count`: states that entered compare but only partial/capped results were available
- the new fields are emitted in planner fuzz summaries/json, campaign aggregate summaries/json, compare-profile summaries, and exhaustive summaries.

12. remaining risks
- production `exact_shadow` is still a bounded policy, not the full tests-only exhaustive canonicalizer.
- `planner_tie_mixed_organic_compare_ready` is intentionally a tests-only split precursor, not a true mixed follow-up execution family. It exists to collect compare evidence without violating the production sole-hosted-occ split precondition.
- compare hot path remains expensive. Even with caches and smaller campaign bounds, exact_shadow lookahead plus exact/full canonicalization dominate runtime on symmetry-heavy states.
