2. Production Split-Choice Policy

Production default split-choice policy is now exact_shadow.

Behavior:
- 0 admissible split pairs: return nullopt
- 1 admissible split pair: choose it directly
- 2+ admissible split pairs: enumerate support-based brute-force candidates, evaluate each pair by bounded lookahead:
  - perform split
  - apply afterSplit follow-up hooks
  - run current planner normalization
  - capture failed / stopConditionSatisfied / target isolate canonical key / final canonical raw state key / failureKey
- choose the lexicographically minimal ProductionSplitChoiceSignature
- original label order is not used as the primary policy key

Fast policy was retained only for comparison / regression reproduction / fallback accounting.
Production default is exact_shadow in include/raw_engine/raw_engine.hpp.

3. Production Canonical Key Design

Production path uses a minimal shared key, not the full tests-only exhaustive canonicalizer.

ProductionSplitChoiceSignature fields:
- failed
- stopConditionSatisfied
- target isolate canonical key
- final canonical raw state key
- failureKey

Ordering:
- lexicographic total order over the fields above
- same-signature ties are resolved deterministically without directly using original labels as the primary rank

Tests-only exact canonicalizer remains in tests/ and is still used for audits.
