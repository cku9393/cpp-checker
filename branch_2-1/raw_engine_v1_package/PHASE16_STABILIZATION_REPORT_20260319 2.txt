3. policy graduation gate definition
- production default remains `exact_shadow`; `exact_full` stays compare/debug/nightly-only.
- graduation threshold table is defined in `policy_graduation_threshold_for_family()`:
  - `split_tie_ready >= 16/16/16`
  - `split_tie_organic_symmetric >= 32/32/32`
  - `planner_tie_mixed_organic_compare_ready >= 32/32/32`
  - `automorphism_probe_large >= 32/32/32`
  - tuple order is `compared_state_count / eligible_state_count / completed_state_count`
- gate evaluation is implemented in `evaluate_policy_graduation_gate()`:
  - `FAIL` if any of `split_choice_semantic_disagreement_count`, `split_choice_fallback_count`, or `semantic_shift_count` is non-zero
  - `INSUFFICIENT_EVIDENCE` if compared/eligible/completed counts are below the effective threshold
  - `PASS` if thresholds are met and `representative_shift_count == 0`
  - `SOFT_PASS` if thresholds are met, representative shifts exist, but `semantic_shift_count == 0`
- CLI/runtime knobs that can raise the effective target:
  - `--target-compared-states`
  - `--target-eligible-states`
  - `--stop-when-gate-passes`
  - `--max-partial-runs`
- aggregate campaign output now emits:
  - `gate_status`
  - `stop_reason`
  - `run_gate=...`
  - `target_compared_states`
  - `target_eligible_states`
  - `compare_completed_state_count`
- smoke coverage:
  - `split_choice_policy_graduation_smoke`
  - `compare_evidence_threshold_smoke`
