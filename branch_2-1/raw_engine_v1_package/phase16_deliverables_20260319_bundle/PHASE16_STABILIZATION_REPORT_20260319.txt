3. policy graduation gate definition
- production split-choice policy remains `exact_shadow`; `exact_full` stays compare/debug/nightly-only.
- new CLI / checkpoint controls:
  - `--target-compared-states`
  - `--target-eligible-states`
  - `--stop-when-gate-passes`
  - `--max-partial-runs`
- gate statuses:
  - PASS
  - SOFT_PASS
  - INSUFFICIENT_EVIDENCE
  - FAIL
- gate rules:
  - FAIL if `split_choice_semantic_disagreement_count != 0`
  - FAIL if `split_choice_fallback_count != 0`
  - FAIL if `semantic_shift_count != 0`
  - PASS if thresholds are met and `representative_shift_count == 0`
  - SOFT_PASS if thresholds are met, representative shifts exist, and `semantic_shift_count == 0`
  - INSUFFICIENT_EVIDENCE otherwise
- thresholds encoded in `tests/raw_engine_cases.cpp`:
  - `split_tie_ready`: 16 / 16 / 16
  - `split_tie_organic_symmetric`: 32 / 32 / 32
  - `planner_tie_mixed_organic_compare_ready`: 32 / 32 / 32
  - `automorphism_probe_large`: 32 / 32 / 32
- phase16 hardening applied while finishing the work:
  - resume now preserves explicit CLI overrides for the new gate controls instead of silently forcing the manifest values
  - completed-state target now tracks the stronger of compared / eligible overrides
  - aggregate writer now prefers persisted per-run summaries when mixed partial runs would otherwise leave in-memory aggregate stats stale
