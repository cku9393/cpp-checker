2. exact_full compare mode design
- Production default split-choice policy remains exact_shadow.
- tests-only exact_full compare reuses the already-evaluated admissible pair outcomes and the tests exact canonical key to compute the exact representative without changing production planner behavior.
- Compare path records:
  - split_choice_compare_state_count
  - split_choice_exact_full_eval_count

- split_choice_fallback_count is now surfaced in split-choice summaries and fuzz aggregation.

3. Multiclass catalog design
- Multiclass states are cataloged by a family-scoped dedupe key:
  family + ":" + category + ":" + base_cluster_key
- This prevents cross-family cluster collapse.
- Each catalog entry now emits:
  - .summary.txt witness
  - .state.txt replayable planner dump
- final_state_divergence is based on the exact-audit projection comparison, not the old fast final-state hash.
